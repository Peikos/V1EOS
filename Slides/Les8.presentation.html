Where to start?

  • BIOS en UEFI in ROM (Firmware), starten met PC
  • Eerst controle hardware (POST), dan zoeken naar bootable disk
  • Op bootable disk MBR (1e sector) met partities, verwijzing bootable partitie
  • 1e sector per partitie (of voor partitie) is partition boot sector
  • BIOS / UEFI
  • Firmware (EEPROM)
  • Power-On Self-Test
  • Boot-disk, MBR en boot-sector

De boot-sector

  • Boot sector = 512 bytes, vrij beperkt
  • Boot Sector signature: laatste bytes 0x55aa
  • Sector komt in vast geheugenadres (0x7c00)

2nd-stage bootloader

  • 512 bytes is vrij weinig
  • 2 trappen: 512 bytes om 2nd stage bootloader te laden
  • Deze kan vervolgens 1 of meerdere OSs starten
  • Grub voor Linux, BOOTMGR voor Windows

Compilation vs intepretation

  • Strategie niet afhankelijk van taal, maar vaak wel “standaard” per taal
  • Interpretatie: als python - source wordt on the fly vertaald
  • Compilen: als C++ - source wordt naar binary vertaald
  • JIT als Java - ertussenin, gedeeltelijke vertaling

V1E-OS: een simpele bootloader

  • Loop kort door code heen, dit is alles wat er nodig is
  • Hoeven ze niet te kunnen
  • Belangrijk take-away point: ook hier geen black magic, alleen speciale code
  • Ook zij zouden dit kunnen maken
bits 16
org 0x7c00              ; begin op dit adres
boot:
    mov si,welcome      ; si bevat het adres van welcome
    mov ah,0x0e         ; nodig voor interupt 0x10: schrijf karakter
.loop:
    lodsb               ; laad eerste karakter in al
    or al,al            ; als al 0 is:
    jz halt             ;   spring naar halt
    int 0x10            ; BIOS interrupt 0x10 (video services)
    jmp .loop           ; loop
halt:
    cli                 ; clear interrupt flag
    hlt                 ; halt execution
welcome: db "V1E-OS loading...",0
times 510 - ($-$$) db 0 ; vul tot byte 510
dw 0xaa55               ; laatste twee bytes: signature

BIOS-calls / Interrupts

  • Als system calls voor software, maar dan OS→hardware
  • int instructie met aanduiding voor categorie
  • rest via registers
  • Function code in AH / BH
  • Parameters in overige registers
  • Return values in registers
  • int 0x10 voor video services
  • int 0x13 voor low-level disk services
  • int 0x15 voor system services

32-bits mode

  • Standaard 16 bits. 32 of 64 bits moet aangevraagd worden
  • Hiervoor GDT nodig, volgende slide
  • Tevens interrupt om meer geheugen aanspreekbaar te maken.
bits 16
org 0x7c00               ; begin op dit adres
boot:
    mov ax, 0x2401       ; function code
    int 0x15             ; enable >1MB geheugen
    cli
    lgdt [gdt_pointer]   ; laad GDT tabel
    mov eax, cr0
    or eax,0x1           ; protected mode bit 1
    mov cr0, eax
    jmp CODE_SEG:boot2
bits 32
boot2:
    ; <32-bits instructies>
    cli                 ; clear interrupt flag
    hlt                 ; halt execution
times 510 - ($-$$) db 0 ; vul tot byte 510
dw 0xaa55               ; laatste twee bytes: signature

De General Descriptor Table

  • Tabel met offsets waar segmenten beginnen en hoe groot deze zijn
  • Binair opgebouwd, letterlijke bytewaardes
  • In assembly db data byte en dw data word etc

Code voor de GDT

Hier een voorbeeld tabel in een fragment assembly

; offset 0x0
.null descriptor:
    dq 0

; offset 0x8
.code:              ; cs register verwijst hiernaar
    dw 0xffff       ; segment limit eerste 0-15 bits
    dw 0            ; base eerste 0-15 bits
    db 0            ; base 16-23 bits
    db 0x9a         ; access byte
    db 11001111b    ; eerste 4 bits (flags) rest (limit 4 laatste bits)(limit is 20 bits breeds)
    db 0            ; base 24-31 bits

; offset 0x10
.data:              ; ds, ss, es, fs, en gs verwijzen hiernaar
    dw 0xffff       ; segment limit eerste 0-15 bits
    dw 0            ; base eerste 0-15 bits
    db 0            ; base 16-23 bits
    db 0x92         ; access byte
    db 11001111b    ; eerste 4 bits (flags) rest (limit 4 laatste bits)(limit is 20 bits breeds)
    db 0            ; base 24-31 bits

Het leven zonder glibc

  • Wel C++, geen standard library (libc)
  • Libc heeft syscalls nodig om functionaliteit te leveren, die zitten in kernel
  • Kernel spreekt direct tot hardware, dus met interrupts
  • Voorbeeld: Loopen door geheugenadressen van scherminhoud om te printen
  • kmain is het startpunt omdat dat in stuk assembly vorige slide staat, kan alles zijn
